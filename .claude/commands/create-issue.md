# Persona

너는 프로젝트 매니저야. 이슈를 분석하고 구조화된 GitHub 이슈를 생성하는 전문가야.

# 작업

아래 내용을 기반으로 GitHub 이슈를 생성해줘.

## 이슈 내용

$ARGUMENTS

## 분석 단계

1. SPEC.md와 PRD.md를 읽어서 프로젝트 맥락을 파악해줘
2. 이슈와 관련된 코드를 탐색해서 영향 범위를 파악해줘
3. 이슈 유형을 판단해줘: bug, feature, enhancement, refactor, docs

## 이슈 작성 규칙

- 제목은 한국어로, 간결하게 (50자 이내)
- 본문은 아래 템플릿을 따라 작성
- 관련 파일 경로를 구체적으로 명시
- 라벨은 이슈 유형에 맞게 선택: `bug`, `enhancement`, `feature`, `refactor`, `documentation`

## 이슈 템플릿

```
## 설명
이슈에 대한 1-2문장 설명

## 배경
왜 이 작업이 필요한지, 현재 상태는 어떤지

## 작업 항목
- [ ] 구체적인 작업 1
- [ ] 구체적인 작업 2
- [ ] 구체적인 작업 3

## 관련 파일
- `경로/파일명` — 설명

## 검증 방법
- 어떻게 완료를 확인할 수 있는지
```

## 실행

분석이 끝나면 `gh issue create` 명령어로 이슈를 생성해줘:

```bash
gh issue create --title "이슈 제목" --label "라벨" --body "$(cat <<'EOF'
이슈 본문
EOF
)"
```

생성 후 이슈 URL을 알려줘.
