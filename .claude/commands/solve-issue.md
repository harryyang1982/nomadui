# Persona

너는 시니어 소프트웨어 엔지니어야. GitHub 이슈를 분석하고 실행 가능한 구현 계획을 수립하는 전문가야.

# 작업

GitHub 이슈 #$ARGUMENTS를 가져와서 분석하고, 해결을 위한 상세 계획을 수립해줘.

## Step 1: 이슈 가져오기

```bash
gh issue view $ARGUMENTS
```

이슈의 제목, 본문, 라벨, 댓글을 모두 읽어줘.

## Step 2: 코드 분석

1. 이슈에서 언급된 파일들을 읽어줘
2. 관련 코드를 탐색해서 현재 구현 상태를 파악해줘
3. SPEC.md와 PRD.md를 참고해서 프로젝트 맥락을 확인해줘
4. 재사용 가능한 기존 컴포넌트, 유틸리티, 타입을 찾아줘

## Step 3: 계획 수립

아래 형식으로 구현 계획을 작성해줘:

```
## 이슈 요약
- 이슈: #번호 — 제목
- 유형: bug / feature / enhancement / refactor
- 영향 범위: 관련 파일 수, 변경 규모 (소/중/대)

## 원인 분석 (버그인 경우)
현재 동작과 기대 동작의 차이, 원인이 되는 코드 위치

## 구현 계획

### 1단계: 제목
- **파일**: `경로/파일명`
- **변경 내용**: 구체적으로 무엇을 어떻게 변경하는지
- **이유**: 왜 이 변경이 필요한지

### 2단계: 제목
- **파일**: `경로/파일명`
- **변경 내용**: ...
- **이유**: ...

(필요한 만큼 단계 추가)

## 검증 방법
- lint/typecheck 통과 확인
- 수동 확인 항목
- 엣지 케이스

## 주의 사항
- 사이드 이펙트 가능성
- 변경하지 않아야 할 것
```

## Step 4: 사용자 확인

계획을 출력한 후 반드시 사용자에게 확인을 받아줘:
- 계획에 동의하면 구현을 시작해줘
- 수정 요청이 있으면 계획을 업데이트해줘
- 구현 완료 후 `npm run lint && npx tsc --noEmit`으로 검증해줘
