## 1. 문제 정의

### 1.1 배경

해외에는 Nomads.com(구 Nomad List)과 같이 디지털 노마드를 위한 도시별 생활 조건을 체계적으로 정리하고 커뮤니티를 제공하는 플랫폼이 존재한다. Nomads.com은 2014년부터 운영되어 40,000명 이상의 회원과 8,300개 이상의 리뷰를 보유하고 있으며, 200개 이상의 데이터 포인트로 전 세계 도시를 평가한다.

그러나 **대한민국에 특화된 디지털 노마드 정보 플랫폼은 현재 존재하지 않는다.** 한국은 세계 최고 수준의 인터넷 인프라, 높은 안전도, 저렴한 대중교통, 풍부한 카페 문화 등 디지털 노마드에게 매력적인 조건을 갖추고 있음에도 불구하고, 도시별 생활 비용, 코워킹 환경, 인터넷 속도, 외국인 친화도 등을 한눈에 비교할 수 있는 서비스가 부재하다.

### 1.2 핵심 문제

| 문제 | 설명 |
|------|------|
| 정보 분산 | 생활비, 인터넷 속도, 안전도, 코워킹 스페이스 등 정보가 블로그, 카페, SNS에 흩어져 있음 |
| 비교 불가 | 도시 간 조건을 동일한 기준으로 비교할 수 있는 플랫폼이 없음 |
| 실사용자 리뷰 부재 | 실제 노마드 체류 경험 기반의 구조화된 리뷰가 없음 |
| 커뮤니티 부재 | 한국 내 디지털 노마드 간 교류할 수 있는 전용 온라인 커뮤니티가 없음 |

## 2. 타겟 사용자

### 2.1 사용자 페르소나

**주요 타겟: 디지털 노마드 (예비 + 현재 활동 중)**

| 속성 | 설명 |
|------|------|
| 연령 | 25~40세 |
| 직업군 | 소프트웨어 개발, 글쓰기, 마케팅, 디자인, 콘텐츠 크리에이터 등 |
| 특징 | 컴퓨터 활용 능력 높음, SNS 활발, 업무 환경을 크게 가리지 않음 |
| 핵심 니즈 | 안정적인 인터넷, 합리적 생활비, 작업 가능한 카페/코워킹, 안전한 환경 |
| 언어 | 한국어 (1차), 영어 (2차 확장) |

### 2.2 사용자 세그먼트

| 세그먼트 | 설명 | 홈페이지에서의 행동 |
|----------|------|---------------------|
| 탐색형 | 노마드 생활을 고려 중, 어떤 도시가 좋은지 리서치 | 도시 카드 브라우징, 필터링, 정렬 |
| 비교형 | 2~3개 후보 도시를 놓고 비교 중 | 비교 뷰 사용, 상세 점수 확인 |
| 리뷰형 | 이미 체류 경험이 있고 정보를 공유하고 싶음 | 리뷰 섹션 확인 후 가입 전환 |
| 커뮤니티형 | 다른 노마드와 교류하고 싶음 | 밋업/채팅 위젯 확인 후 가입 전환 |

## 3. 제안 해결책

### 3.1 제품 개요

**KoNomad**는 대한민국 50개 이상 도시의 디지털 노마드 생활 조건을 실시간 데이터와 사용자 리뷰 기반으로 비교할 수 있는 웹 플랫폼이다.

홈페이지는 플랫폼의 첫인상이자 핵심 전환 포인트로, 다음 두 가지 목표를 동시에 달성해야 한다:

1. **정보 제공**: 비로그인 상태에서도 도시 카드 브라우징, 필터링, 기본 점수 확인 가능
2. **전환 유도**: 상세 정보 접근, 리뷰 작성, 커뮤니티 참여를 위한 회원 가입 유도

### 3.2 경쟁 분석 (Nomads.com 벤치마킹)

Nomads.com 실사 분석을 통해 도출한 핵심 참고 요소:

| Nomads.com 특징 | KoNomad 적용 방안 |
|-----------------|-------------------|
| 200개+ 정렬 옵션 | 한국 특화 15개 핵심 지표로 간소화 |
| 4열 그리드 (3카드 + 1사이드바) | 동일 구조 채택 |
| 이모지 기반 직관적 데이터 표현 | 동일 방식 적용 |
| 하단 고정 CTA 바 | 동일 패턴 적용 |
| Social Proof (언론 로고, 멤버 수, 리뷰 수) | 동일 전략 적용 |
| 사이드바 위젯 (밋업, 여행중 멤버, 신규 멤버) | 동일 구조 적용 |
| 도시 카드에 실시간 날씨/AQI 표시 | 기상청 API 연동으로 적용 |
| 비디오 배경 히어로 | 한국 도시 풍경 영상으로 적용 |

#### [S6-1] 도시 카드

**레이아웃**: 3열 그리드 + 1열 사이드바

**카드 구성 요소**:

| 요소 | 데이터 타입 | 설명 |
|------|------------|------|
| 순위 | 숫자 | 현재 정렬 기준 순위 (#1, #2...) |
| 도시 이미지 | 이미지 | 도시 대표 풍경 사진 (배경) |
| 인터넷 속도 | 숫자 + 단위 | 📶 XXX Mbps |
| 도시명 | 텍스트 | 한글명 (영문명) |
| 지역 | 텍스트 | 대한민국 · 수도권/영남권/... |
| 현재 날씨 | 이모지 + 숫자 | ☀️ XX°C + 체감 온도 |
| 대기질 | 이모지 + 숫자 | AQI XX + 상태 이모지 |
| 월 생활비 | 통화 | ₩X,XXX,XXX / 월 |
| 노마드 점수 | 별점 | ⭐ X.X/5 |
| 리뷰 수 | 숫자 | 리뷰 XXX개 |
| 좋아요/싫어요 | 숫자 | 👍 XXX 👎 XX |

| 영역 | 기술 | 선택 이유 |

| 인증 | Supabase Auth | Google, Kakao 소셜 로그인 |
| 지도 | Kakao Maps API | 한국 지도 최적화 |

## 9. 제약 사항 및 가정

### 9.1 제약 사항

| 항목 | 설명 |
|------|------|
| 초기 데이터 | 런칭 시 50개 도시의 정적 데이터로 시작. 사용자 리뷰가 쌓이면서 데이터 품질 향상 |
| 실시간 데이터 | 기상청 API 호출 제한이 있으므로, 30분~1시간 캐싱 적용 |
| 인터넷 속도 | 자체 측정 불가. Speedtest/Ookla 공개 데이터 또는 사용자 제보 기반 |
| 생활비 | 초기에는 리서치 기반 추정치. 사용자 리뷰로 보정 |

### 9.2 가정

| 가정 | 검증 방법 |
|------|-----------|
| 한국에서 디지털 노마드 수요가 충분하다 | 런칭 후 MAU 추적 |
| 사용자가 도시 리뷰를 자발적으로 작성한다 | 리뷰 작성률 추적 |
| 도시 카드 UI가 직관적으로 이해된다 | Hotjar 히트맵 + 사용자 테스트 |
| 무료 기능만으로 첫 달 1,000 MAU 달성 가능하다 | GA4 트래픽 분석 |
